# AI Service Frontdesk 页面结构规划

## 1. 重构目标

状态：未完成

当前 Streamlit 页面把商家配置、客户聊天、知识库状态、当前线索和历史线索放在同一个页面，适合开发调试，但不适合真实使用。

本轮页面重构目标：

1. 将客户咨询入口和商家后台拆开。
2. 将低频配置从左侧栏移到独立页面。
3. 为知识库提供明确入口，可查看文档和测试检索。
4. 保留当前 LangGraph、RAG、storage 等后端能力，优先重构页面结构，不重写核心业务逻辑。
5. 让客户端简单，商家后台清晰，开发调试信息单独隔离。

## 2. 信息架构

状态：未完成

推荐结构：

```text
AI Service Frontdesk

```

## 4. 页面规划

### 4.1 首页

状态：未完成

文件：

```text
app.py
```

目标：

- 作为轻量入口页。
- 告诉使用者当前有哪些页面。
- 不承载核心业务操作。

页面内容：

- 产品名：AI Service Frontdesk
- 当前版本状态
- 快捷入口说明：
  - 客户咨询
  - 线索看板
  - 知识库管理
  - 商家资料
  - 系统测试

不做：

- 不放大段营销文案。
- 不放商家配置表单。
- 不放完整聊天窗口。
- 不放调试 JSON。

验收标准：

- 打开应用时页面清爽。
- 用户能从侧边导航进入各功能页。

### 4.2 客户咨询页

状态：未完成

文件：

```text
pages/1_Customer_Chat.py
```

目标：

这是客户真正使用的入口，只保留咨询体验。

核心功能：

- 展示品牌名称。
- 展示客户聊天窗口。
- 支持客户输入问题。
- 调用 `run_frontdesk_turn(...)`。
- 展示 AI 回复。
- 自动保存生成或更新的线索。
- 若触发人工接管，显示简洁提示。

可回答问题：

```text
你们公司地址在哪？
王师傅能修水管吗？
深度保洁多少钱？
维修后保修多久？
我家厨房水管漏水，明天上午能来修吗？
```

页面不显示：

- 商家配置表单
- 知识库技术状态
- `retrieved_context`
- 原始 JSON
- 历史线索表
- 调试信息

客户侧提示：

- “最终价格和上门时间需工作人员确认。”
- “已帮您登记，工作人员会尽快联系。”

验收标准：

- 客户打开页面只看到聊天相关内容。
- 客户知识类问题可直接得到回答。
- 客户询盘类问题会被追问缺失字段。
- 客户补充电话和地址后会生成线索。
- 投诉场景会提示人工接管。

### 4.3 线索看板页

状态：未完成

文件：

```text
pages/2_Leads_Board.py
```

目标：

商家查看和跟进客户线索。

核心功能：

- 展示所有线索。
- 展示线索状态。
- 支持按状态筛选。
- 展示关键字段：
  - 状态
  - 服务需求
  - 联系电话
  - 服务地址
  - 期望时间
  - 紧急程度
  - 报价草稿
  - 更新时间

状态显示：

| 内部状态 | 页面显示 |
|---|---|
| `new` | 新线索 |
| `needs_info` | 待补充信息 |
| `quoted` | 已生成报价 |
| `handoff_required` | 需人工接管 |
| `closed` | 已关闭 |

可选功能：

- 清空历史线索。
- 刷新线索列表。
- 展开单条线索详情。

暂不做：

- 登录权限。
- 多人协作。
- 在线修改状态。
- 导出 CSV。

验收标准：

- 能从 `data/leads.json` 读取线索。
- 刷新页面后线索仍展示。
- 投诉线索显示为“需人工接管”。
- 报价线索显示为“已生成报价”。

### 4.4 知识库管理页

状态：未完成

文件：

```text
pages/3_Knowledge_Base.py
```

目标：

给商家和运营查看企业知识库资料，并测试 RAG 检索。

核心功能：

- 展示知识库状态：
  - 是否可用
  - Markdown 文件数量
  - 索引文件位置
  - 当前检索策略
  - 是否启用本地 embedding
  - 是否启用 OpenAI embedding
- 展示 `knowledge/` 下所有 `.md` 文件。
- 点击或选择文件后查看文档内容。
- 支持重建知识库索引。
- 支持输入测试问题并显示检索结果。

测试问题示例：

```text
你们公司地址在哪？
王师傅能修水管吗？
深度保洁多少钱？
维修后保修多久？
```

检索结果展示：

- 来源文件
- 标题
- 分数
- 命中的片段内容

暂不做：

- 文件上传。
- 在线编辑 Markdown。
- 多版本知识库。
- 权限控制。

验收标准：

- 能看到 `company.md`、`pricing.md`、`staff.md`、`service_policy.md`、`faq.md`。
- 能查看每个文档内容。
- 点击“重建知识库索引”后成功完成。
- 输入测试问题后能显示命中片段。

### 4.5 商家资料页

状态：未完成

文件：

```text
pages/4_Business_Profile.py
```

目标：

承载低频商家基础配置，替代当前左侧冗余配置区。

核心功能：

- 编辑商家名称。
- 编辑服务行业。
- 编辑服务区域。
- 编辑营业时间。
- 编辑服务项目。
- 编辑默认价格规则。
- 编辑 FAQ。
- 保存到 `data/business.json`。

注意：

- 这是后台配置页，不应该出现在客户咨询页。
- 页面可以简洁，不需要频繁打开。
- 后续 RAG 价格资料和商家基础资料需要明确边界：
  - `business.json` 用于默认配置。
  - `knowledge/*.md` 用于企业知识检索。

验收标准：

- 能读取当前商家配置。
- 能保存修改。
- 保存后客户咨询页使用最新配置。

### 4.6 系统测试页

状态：未完成

文件：

```text
pages/5_System_Test.py
```

目标：

给开发者和运营测试 AI 回复、线索抽取和 RAG 命中。

核心功能：

- 输入测试问题。
- 调用 `run_frontdesk_turn(...)`。
- 展示 AI 回复。
- 展示结构化线索字段。
- 展示缺失字段。
- 展示是否人工接管。
- 展示 `retrieved_sources`。
- 展示 `retrieved_context`。

适合测试：

```text
你们公司地址在哪？
我家厨房水管漏水，明天上午能来修吗？
地址在浦东新区张江花园小区，电话13800138000
刚才维修师傅把我家弄坏了，我要投诉。
```

页面不面向客户。

验收标准：

- 测试问题后能看到完整调试信息。
- RAG 命中来源清晰。
- 线索抽取字段清晰。

## 5. 导航与操作逻辑

状态：未完成

客户路径：

```text
客户咨询页 -> 输入问题 -> AI 回答/追问 -> 补充信息 -> 生成线索
```

商家跟进路径：

```text
线索看板 -> 查看新线索 -> 人工联系客户 -> 后续处理
```

商家维护资料路径：

```text
知识库管理 -> 查看 Markdown -> 重建索引 -> 测试检索
```

商家基础配置路径：

```text
商家资料 -> 修改基础配置 -> 保存
```

开发/运营调试路径：

```text
系统测试 -> 输入测试问题 -> 查看 AI 回复、线索字段和 RAG 来源
```
